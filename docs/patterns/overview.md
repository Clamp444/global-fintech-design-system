# Patterns

Patterns are reusable page-level and flow-level structures. They sit above individual components and below full case studies.

Use this section when a design needs a repeatable product structure, such as a dashboard, onboarding flow, list page, form flow, or payment journey.

## Pattern Layers

| Layer | Description | Example |
|---|---|---|
| Component | A reusable UI building block | Button, Input, Metric Card |
| Pattern | A reusable page or flow structure | Dealer Dashboard, Dealer Onboarding |
| Case Study | A complete product or brand story | PalmPay Brand System |

## Rules

- A pattern must list the components it depends on.
- A pattern must define required states, not only the ideal state.
- A pattern must include localization and multi-market risks when relevant.
- A pattern must be specific enough for AI to generate a Figma draft without inventing structure.
